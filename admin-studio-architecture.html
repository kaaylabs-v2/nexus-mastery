Admin Studio Screens

1. Dashboard

Overview: total learners, active courses, avg mastery score, enrollment trends. Quick actions: upload new course, invite users. Recent activity feed.

2. Course Upload / Creator

Drag & drop upload zone. Supports PDF, DOCX, TXT, MD, PPT, CSV. Shows real-time processing pipeline (extracting → analyzing → generating → done). AI auto-generates course metadata. Admin reviews and edits before publishing.

3. Courses List

All courses for this org. Status: draft / active / archived. Shows enrolled learners count, avg mastery level, content sources. Search, filter, sort. Bulk actions.

4. Course Detail / Editor

Edit course metadata (title, description, mastery criteria). View/manage uploaded materials. See content chunks. Add more materials to existing course. Preview how Nexi will use this content.

5. Users & Enrollment

All users in this org. Invite via email or SSO sync. Roles: learner / facilitator / org_admin. Bulk enroll into courses. See enrollment status per user (NO mastery profile access — privacy enforced).

6. Analytics

Org-level aggregate stats only: enrollment completion rates, avg mastery progression, course effectiveness. No individual mastery profiles. Heatmap of active learning times.

7. Org Settings

Organization name, branding (logo, primary color). SSO/SAML configuration. Plan tier. API keys for integrations. Webhook endpoints. Notification preferences.

8. Invite / Onboarding

Bulk CSV import of users. Email invitation with org branding. SSO auto-provisioning (SCIM). Welcome message customization.

Multi-Tenancy Enforcement
Data Isolation
• Every query filters by org_id from JWT token
• File storage paths: /uploads/{org_id}/{course_id}/
• Embeddings scoped by course_id → org_id
• RAG retrieval only pulls from tenant's courses
• No cross-tenant data access possible at DB layer
Privacy Boundaries
• Admin sees: enrollment status, completion %, course stats
• Admin CANNOT see: thinking_patterns, knowledge_graph, conversation_summary, pacing_preferences
• Analytics are aggregates only — no individual profiles
• Enforced at API layer AND at DB query level
• Privacy tests extended for admin routes
